HTML Tags:

Paragraph: "< p >"
  • Must have an Opening tag, and usually a Closing tag, and surround some content.
  • "p" by itself is an element, but is interchangablely used with tag.
Some tags are only opening tags such as:
  • Line Break: "< br >"
  • Horizontal Rule: "< hr >"
Every HTML element has predefined attributes.
Attributes are a name value pair that is a meta data for the element it is assigned to.
No other element on the webpage is allowed to have that id attribute.
Example:
  • "< p id="myID"> < /p >"
  • id is the Attribute Name
  • myID is the Attribute Value
No spaces between the opening/ending bracket and the element name. Spaces inside are ignored.

Quotes within quotes: both single quotes (') and double quotes (") can be used, and can be nested, as long as they are closed.

"Self-closing" tag: in other versions of HTML, there were "self-closing" tags that could be used as empty place holders to dynamically insert data later on. "< p/ >"

However, in HTML5, "self-closing" is illegal. Instead, just have the opening and closing tags with no space in-between to create the same affect. "< p >< /p >"

Basic HTML Document Structure:

"< !doctype html >"
Every HTML page should begin with the "< !doctype html >" declaration.
  • This can be written in lower or upper case. No spaces between the "<" symbol and "!"
  • This is a necessary part since it tells the browser that your page is following standard HTML declaration.
"< html >< /html >"
Next is the html tag that encompasses the entire html document.

"< head >< /head >"
Then the head tag, which contians the main content of the page such as the page title, author description, and etc.
    "< meta charset="utf-8" >"
  • Inside the head tag, is the meta tag, to specify the character set. This will let the browser know how to interpret the content of the webpage. Commonly used is UTF-8.
  • It is a stand-alone tag.

    • "< title >. . .< /title >"
  • The title tag is a necessary tag. Without it, the HTML code would be invalid.
"< body >. . ."
After the head tag is the body tag. This is the root of all content that is visible by the user (aka the viewport).

**Important Rule: You must close the last opened tag before closing its parent's tag. Or else it the html is invalid.

HTML is read from the top down sequentially by the browser.

HTML Content Models:

Content Model -- the full behavior the browser applies to the elements within the content model, and which elements are allowed to be nested inside which elements.

Generally split into 2 categories:
  • Block-Level Elements
  • InLine Elements
HTML5 split these categories into 7 different models.

Block-Level Elements:
  • The browser automatically places the element on a new line in the flow of the document (by default)
  • May contain inline or other block-level elements within the element
Inline Elements:
  • The browser automatically places the element on the same line (by default), as if there was no new line character present
  • May only contain other inline elements
Although HTML5 replaces the 2 categories, the distinction between elements is still practical since it aligns with current existing CSS rules.

"< div >"
The div tag stands for a division. It is your most generic block level element.

"< span >"
The span tag stands for span. It is your most generic inline element.

Semantic HTML Elements:

Semantic -- relating to meaning in lnaguage or logic
Semantic HTML Element -- an element that implies some meaning to the content

Heading Example:

MAIN HEADING 1

SUBHEADING 2

SUBHEADING 3

This shows that the most important headings start from h1. This will make it easier for SEO (Site Engine Optimization)
The following tags are all block-line elements.

"< header >"
Contains header information about the page. Perhaps logos, navigation, etc.

"< nav >"
Usually contains links to different parts of the web site to navigate.

"< section >"
Acts like a div tag. Generally holds inline elements such as article tags.

"< article >"
Acts like a span tag. However, it can hold section tags.

"< aside >"
Contains some information that is generally related to the main topic of the site.

"< footer >"
Contains the footer information. Perhaps copyright and contact info.

These types of semantic elements allow for more meaningful breakdowns of an html page. However they do not add much functionality when compared to "< div >" and "< span >" tags.

HTML Lists

"< ul >"
Represents an unordered list tag. Every element within the ul tag is an "li" tag (list item).
Example:
  • Milk
  • Donuts
  • Cookies
    • Chocolate
    • Sugar
    • Peanut Butter
  • Pepto Bismol
"< ol >"
Represents an ordered list tag. Every element within the ul tag is an "li" tag (list item).
Example:
  1. Milk
  2. Donuts
  3. Cookies
    1. Chocolate
    2. Sugar
    3. Peanut Butter
  4. Pepto Bismol

3 Characters You Should Always Escape

To let the computer differentiate which characters are part of the HTML syntax and which characters are actually part of the desired information.

" < "
Instead of the less than symbol use: "<"
" > "
Instead of the less than symbol use: ">"
" & "
Instead of the less than symbol use: "&"

Other important HTML Character Entity References

Copyright Symbol:
The copyright symbol can be found with: "©"
Non-breaking Space
To add a non-breaking space (to prevent certain words from being split apart when viewing) use: "word1 word2 word3"
Quotes
To add quotes (useful when using other limited character sets) use: " " "

Creating Links

To make links, you must first specify an anchor tag, "a" element (<a>) with an attribute of "href" which stands for hypertext reference.
  • The value of href can either be a relative or an absolute URL.
  • Internal links are all links that point to internal web pages of the web application.
You should also specify a "title" attribute for the a tag. This title attribute will be used to help visually impaired people get through the web page.

The content between the a tags are the actual information that you will be able to see in the web page and that you can click.

Internal Links:

You can have links inline.
For Example: Linking to a file within the "test_site" folder on this machine.

You can also have the link onto a new line by using a div tag around the content that you want to click. The reason why this works is because the a tag is actually both an inline and a block-level element.
For Example:
Linking with DIV to a file within the "test_site" folder on this machine.

External Links:

The href values in external links usually begin with "http://" because the external links are usually hosted on a different domain other than your website.
Addition to the href and title attributes, a tags that lead to external links also have a "target" attribute. The target attribute, when set to "_blank" forces the browser to open in a new tab or window.

  • You can create an anchor tag with a name attribute (named similarly to section names).

Displaying Images

HTML Images:

Use the "img" tag (<img&mt;) to display images. It is an inline element.
There are 4 important attributes you should include in an img tag
  1. "src" tag
    • A source tag is similar to an href tag since the url points to the location of an image. This url can be relative or absolute url or even an external url.
  2. "width" tag
    • Specifies how wide an image is in pixels.
    • Tells the page how many pixels to allocate even if the image takes a long time to load. (This makes the page more visually appealing especially if the image is an external URL)
  3. "height" tag
    • Specifies how tall an image is in pixels.
    • Tells the page how many pixels to allocate even if the image takes a long time to load. (This makes the page more visually appealing especially if the image is an external URL)
  4. "alt" tag
    • An alternate text tag adds text to an image's description to help people with visual impairment.
